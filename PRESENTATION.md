## Слайд 8. Бэкенд

Backend написан на Go и выполняет роль центрального REST API. Вся бизнес-логика доступа к данным вынесена в слой repository — handlers не пишут SQL напрямую, а вызывают методы вроде GetPage, Create, FindByEmail. Это упрощает поддержку и тестирование.

Сервер стартует из main.go: подключается пул pgx к PostgreSQL, создаются репозитории для пользователей, объектов и избранного, регистрируются handlers. Роутинг на chi: публичные маршруты доступны без токена — просмотр каталога с пагинацией, прогноз цены, аналитика. Защищённая группа обёрнута в middleware AuthMiddleware: без заголовка Bearer JWT запрос отклоняется с 401.

При регистрации пароль не хранится в открытом виде — bcrypt.GenerateFromPassword. При логине сравнивается хеш, выдаётся JWT с claim user_id и сроком 24 часа.

Список объектов: GET properties принимает page, limit и city. В репозитории сначала COUNT, затем SELECT с LIMIT и OFFSET — на фронт уходит 20 записей и метаданные для перелистывания страниц.

Прогноз: Go не считает цену сам — валидирует входные поля area, rooms, city и проксирует запрос в Python ML-сервис. Так фронтенд зависит только от одного адреса — :8080.

После ручного добавления объекта Go в горутине вызывает ML /retrain, чтобы модель обновилась на новых данных, не блокируя ответ пользователю.

Дополнительно: CORS для localhost:5173, graceful shutdown по SIGTERM — корректная остановка в Docker.

# Возможные вопросы комиссии

**Почему Random Forest, а не нейросеть?**  
> Для табличных данных с категориями Random Forest даёт хорошее качество при меньшей сложности, быстрее обучается и проще объяснить на защите.

**Откуда данные?**  
> Парсер ЦИАН (cianparser) по 26 городам + ручной ввод + скрипт демо-наполнения для тестирования ML.

**Что именно предсказывает ML?**  
> Только цену объекта по признакам. График на 12 месяцев — формула от справочного % роста региона.

**Как оценивается качество модели?**  
> MAE, RMSE, R² на отложенной выборке 20%; при 2600 объектах R² ≈ 0.94.

**Почему микросервисы?**  
> Разделение ответственности: Go — API и безопасность, Python — ML и парсинг, независимое обновление и масштабирование.

**Как защищены пароли?**  
> bcrypt-хеш в PostgreSQL, JWT в заголовке Authorization.

**Пагинация — где реализована?**  
> SQL LIMIT/OFFSET в Go-репозитории, параметры page и limit в GET /api/properties.
