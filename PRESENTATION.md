## Слайд 2. Актуальность темы

**На слайде (текстом):**

Цена квартиры зависит от города, района, площади, года постройки и других параметров, а на сайтах объявлений (ЦИАН, Авито) сложно быстро понять, соответствует ли цена рынку. Сравнение вручную или в Excel отнимает много времени. Поэтому актуально веб-приложение, которое собирает объявления, хранит их в базе и с помощью машинного обучения прогнозирует стоимость и динамику цен по регионам.

**Речь (≈30 сек):**
> Актуальность темы в том, что рынок жилья неоднороден, а обычные агрегаторы не дают прогноза и единой аналитики. Наше приложение объединяет каталог по 26 городам, фильтры и ML-прогноз в одном интерфейсе.

## Слайд 10. Как работает ML-модель

**На слайде:** схема пайплайна + 4 шага (кратко, без перегруза).

**Схема (вставить на слайд или нарисовать блоками):**

```
  ┌──────────────┐     ┌─────────────────────────────────────────────────┐
  │ PostgreSQL   │────►│ 1. ОБУЧЕНИЕ (при старте / POST /retrain)        │
  │ ~10 000      │     │    • выборка properties                       │
  │ объявлений   │     │    • 5 числовых + 9 категориальных признаков    │
  └──────────────┘     │    • OneHotEncoder + Random Forest (200)        │
                       │    • целевая переменная: log(цена)              │
                       │    • train/test 80/20 → R², MAE, RMSE           │
                       │    • сохранение → price_model.joblib            │
                       └─────────────────────────────────────────────────┘

  Форма «Прогноз» ──► Go /api/predict ──► FastAPI /predict
                                              │
                       ┌──────────────────────▼──────────────────────┐
                       │ 2. ИНФЕРЕНС                                     │
                       │    • подстановка пропусков (медианы из обуч.)   │
                       │    • Random Forest → базовая цена               │
                       └──────────────────────┬──────────────────────┘
                                              ▼
                       ┌─────────────────────────────────────────────────┐
                       │ 3. КОРРЕКЦИЯ (price_adjustments.py)             │
                       │    множители: район, ремонт, этаж, застройщик,  │
                       │    тип жилья (квартира / студия / апартаменты)  │
                       └──────────────────────┬──────────────────────────┘
                                              ▼
                       ┌─────────────────────────────────────────────────┐
                       │ 4. АНАЛИТИКА (analytics.py)                     │
                       │    сравнение с средней по городу, тренд,        │
                       │    прогноз цены на 12 месяцев для дашборда      │
                       └─────────────────────────────────────────────────┘
```

**Признаки модели (одна строка под схемой):**
площадь, комнаты, этаж, этажность дома, год постройки · город, округ, метро, тип дома, застройщик, тип жилья, первичка/вторичка, ремонт квартиры и дома.

**Почему log(цена):** сглаживает разброс цен (от студии до элитного жилья); на выходе — обратное преобразование `expm1`.

**Речь (≈50 сек):**
> ML работает в два режима. При обучении сервис читает все объекты из PostgreSQL, готовит таблицу признаков: числа идут как есть, категории — через One-Hot. Целевая переменная — логарифм цены, чтобы модель лучше училась на объектах разного ценового диапазона. Random Forest обучается на 80% выборки, качество проверяется на 20%; модель сохраняется в файл. При прогнозе пользователь заполняет форму на сайте — запрос идёт в Go и дальше в Python. Модель выдаёт базовую цену, затем мы умножаем её на коэффициенты: например, евроремонт и центр округа дороже, черновая отделка — дешевле. Отдельно считается аналитика: средняя по городу и график на год вперёд по региональному темпу роста. Переобучение — при старте контейнера и по кнопке `/retrain`, когда в базе накопилось много новых объявлений.
